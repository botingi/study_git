git config --global core.autocrlf input
--> OS마다 줄바꿈에 다른 문자열이 입력됨 (ex. windows - /r/n, mac - /n)
    따라서 서로 다른 OS에서 Git에 접근한다면 줄바꿈 문자열로 수정되지 않은 내용들도 
    수정으로 확인되거나 하는 오류가 발생할 수 있다.
    (windows) git config -- global core.autocrlf true 명령어를 입력하게 된다면 
    commit할때, pull할때 /r 문자열을 모두 삭제한다.
    (mac) 반대로 위와 같이 git config --global core.autocrlf input 명령어를 입력한다면
    commit할때만 /r 문자열을 삭제(email이나 다른 문서에서 가져온 /r문자열이 있을 수 있으므로)


git init
--> git 저장소에 등록할 directory위치에서 해당 명령어를 사용함
    사용한 directory에 git에 관련된 파일들이 자동으로 설치됨
    (git 관련 파일들을 삭제하여 git project에서 제거하고 싶다면 rm -rf .git)

Git Workflow
--> Git에서는 [working directory]-[staging area]-[git directory] 세가지의 작업환경으로 나뉜다.
                (수정) ----------> (이동) ----------->(commit)
                (내려받기) <------------------------- (check out)
    다음과 같은 작업은 git history인 local에만 저장되기 때문에 별도의 git server(ex. git hub)에 저장을 하게된다.
             [git directory] --------------> [git server(remote)]
                                  (push)
                             <--------------
                                  (pull)

git add [file]
--> [working directory]에서는 untracked file / tracked file로 나뉜다.
                            그리고 tracked file은 modified/unmodified로 나뉜다.
    git add 명령어를 사용하여 git에서 해당 파일을 tracking할 수 있게 하고, modified file은
    [working directory]에서 [staging area]로 이동하게 된다.
    
git rm --cached [file]
--> [staging area]에 올라간 파일들을 다시 [working directory]로 이동할 수 있다.

echo [file(ex. *.log)] > .gitignore
--> git에서 tracking하는 것을 원치 않는 파일들은 다음 명령어로 제외할 수 있다.

git diff / git difftool
--> git diff 명령어는 terminal 내에서 modified file의 내용을 확인할 수 있다.
    git diff는 config file을 통해 tool을 vscode로 설정한다면, vscode에서 diff내용을 확인할 수 있다.

git commit
--> commit 시 message는 -m명령어를 통해서 간단하게 바로 입력할 수 있다.
    -m 명령어가 없다면 해당 git project에서 생성된 .git 폴더 내 COMMIT_EDITMSG를 수정하여 message를 남길 수 있다.

Branch 기능
--> 하나의 파일을 여러명의 개발자들이 동시 수정이 이루어질때, 각각의 수정된 ver.을 나눠서 작업을 한다. 그리고 수정이 완료된 후 
    두 사람이 수정한 파일 두개를 합쳐야 한다. 이런 상황에서 유용한 기능이 Git의 branch 기능이다.

git branch [branch name]
--> 새로운 branch를 생성할 수 있다. (git을 생성하는 순간 기본 branch는 master)

git checkout [branch name]
--> branch를 변경함. (현재 [git directory] 내용 그대로 [working directory]로 복사)

git log --branches --decorate --graph (--oneline)
--> 모든 branch의 log를 나타낼 수 있음

git log [branch1]..[branch2]
--> [branch1]에서는 생략된 [branch2]에서 생성한 commit을 확인할 수 있다.

git difftool [branch1]..[branch2]
--> [branch1]에서 수정되지 않은 [branch2]에서 추가된 code를 비교해서 확인할 수 있다.

git merge [branch2]
--> 현재 branch1을 접근 중일때, branch2의 수정된 내용을 병합할 수 있다.

Git-hub에 UPLOAD하기
--> git-hub에 연결하는 방법은 두 가지가 있다.
    1. 바로 git-hub 저장소에 내 local directory를 연결하여 바로 upload하는 방법
    2. local저장소에서 작업을 한 후 git-hub저장소에 upload하는 방법 (*)

git remote add origin [git-hub repository address]
--> [git-hub repository]와 local directory를 연결해준다. origin은 해당 git-hub repository의 alias라고
    생각하면 된다. (git remote -v 명령어를 사용하여 repository와 연결된 alias들을 확인할 수 있다.)

git push -u origin main
--> main branch로 git-hub 저장소와 내 local directory를 연결하여 upload해준다.
    (git-hub에서는 기본 default branch name을 master-->main으로 바꿔서 사용하고 있다.)
    (따라서, git branch -M main 명령어로 main이라는 새로운 branch이름을 명명해준다.)
    (With a -m or -M option, <oldbranch> will be renamed to <newbranch>. 
    If <oldbranch> had a corresponding reflog, it is renamed to match <newbranch>, 
    and a reflog entry is created to remember the branch renaming. If <newbranch> exists, 
    -M must be used to force the rename to happen. - http://git-scm.com/docs/git-branch)
